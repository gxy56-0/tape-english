================================================================================
  v2.5 — 主题系统重写 + 取色器自动配色
================================================================================

1. 修复主题切换不生效的 Bug
   - 原因: document.documentElement.style.setProperty() 在部分浏览器中优先级
          低于样式表中的 :root 声明
   - 解决: 改为在 <head> 中动态注入 <style id="theme-style"> 标签，
          用 textContent 覆写 :root 变量，利用 CSS 层叠顺序确保生效

2. 保留 6 套预设主题 (presetThemes)
   - Cyberpunk (青蓝霓虹)
   - Matrix    (荧光绿)
   - Sunset    (暖橙暗金)
   - Ocean     (深海蓝)
   - Sakura    (樱花粉)
   - Minimal   (极简黑白)

3. 新增取色器 + 自动配色系统
   - 添加七彩渐变圆点 (theme-dot-picker)，内嵌 <input type="color">
   - 用户选择任意颜色后，HSL 算法自动生成全套 12 色配色方案:
      背景色 (bg):          同色相, 18%饱和度, 5%亮度
      表面色 (surface):     同色相, 25%饱和度, 9%亮度
      替代表面 (surfaceAlt):同色相, 30%饱和度, 15%亮度
      主强调 (cyan):        用户选择的颜色
      品红 (magenta):       色相 +30°
      紫色 (purple):        色相 +80°
      绿色 (green):         色相 +140°
      红色 (red):           色相 +190°
      琥珀 (amber):         色相 +45°
      文字 (text):          同色相, 8%饱和度, 90%亮度
      弱文字 (muted):       同色相, 12%饱和度, 50%亮度
      边框 (border):        同色相, 22%饱和度, 16%亮度
   - 新增函数: hexToHsl(), hslToHex(), generateThemeFromColor(), applyCustomColor()
   - 自定义主题保存到 localStorage (typing_custom_theme / typing_custom_hex)

4. 顶栏新增 T 按钮
   - 点击在 6 套预设主题间循环切换 (cycleTheme)

5. 更新 init() 逻辑
   - 启动时从 localStorage 恢复自定义主题和取色器值
   - 排除 .theme-dot-picker 避免重复绑定点击事件

================================================================================
  v2.1 — 多词条目空格 + 粤语发音 + 手动播放
================================================================================

1. 多词条目空格跳过逻辑
   - 问题: "cha chaan teng" 需要按空格才能匹配
   - 新增辅助函数:
       typableLength(word)      — 去掉空格后的可打字字符数
       nthNonSpace(word, n)    — 第 n 个非空格字符在原词中的索引
       nonSpaceCountBefore(w,i) — i 位置之前的非空格字符数
   - renderWord(): 空格字符渲染为 .letter.space 类 (视觉保持间距)
   - handleKeydown(): 用 nthNonSpace 映射用户输入到单词位置
   - handleCorrectLetter(): 用 typableLength 判断单词完成
   - updateLetterStates(): 空格跳过颜色判断
   - 启动按键处理同样适配

2. 粤语发音
   - speakCantonese() 函数
   - 优先 Web Speech API zh-HK (Windows: Microsoft Tracy)
   - 兜底 Google Translate TTS (tl=zh-HK)
   - speakWord() 重构: 英语朗读结束后 onend 自动链式调用 speakCantonese
   - zh 字段中 "/" 替换为空格后再朗读

3. 手动发音按钮
   - 位置: 单词显示和进度条之间 (.pronounce-row)
   - EN 按钮: speakEnglish() — 播放英语发音
   - 粤 按钮: speakCantonese() — 播放粤语发音
   - 不受静音开关影响
   - speakEnglish/speakCantonese 均加 speechSynthesis.cancel()

================================================================================
  v2.0 — 大小写不敏感 + 自动发音 + 单词列表
================================================================================

1. 大小写不敏感匹配 (3 处)
   - updateLetterStates: userInput[idx].toLowerCase() === word.en[idx].toLowerCase()
   - handleKeydown: e.key.toLowerCase() === word.en[idx].toLowerCase()
   - 启动首字母: e.key.toLowerCase() === word.en[0].toLowerCase()

2. 自动发音 (Web Speech API)
   - speakWord(): SpeechSynthesisUtterance, lang="en-US", rate=0.85
   - 每次 renderWord 时自动朗读
   - 顶栏 S 按钮切换静音 (toggleSound)
   - 兼容性: window.speechSynthesis 不可用时静默跳过

3. 单词列表 + 学习统计
   - localStorage 持久化: typing_word_stats
   - 每个词记录 attempts (尝试次数) 和 completed (完成次数)
   - renderWord() 中 recordAttempt(), advanceWord() 中 recordCompletion()
   - 顶栏 W 按钮打开单词列表面板 (.wordlist-overlay)
   - 表格: Word / Translation / Category / Attempts / Completed
   - 点击表头排序 (wordListSort)
   - 关闭方式: X / 背景点击 / Escape / 按字母键
